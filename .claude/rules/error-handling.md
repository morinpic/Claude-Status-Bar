# エラーハンドリング

## 基本方針
- `try!` / `as!` は原則使わない
- エラーは適切な型で定義し、呼び出し元に伝播させる
- ユーザーに見せるエラーと開発者向けエラーを区別する

## エラー型の定義
- ドメインごとに Error プロトコルに準拠した enum を定義する
- `LocalizedError` に準拠してユーザー向けメッセージを提供する
- エラーケースは具体的に命名する（`.failed` のような曖昧な名前を避ける）

## Result 型
- async/await が使えない場面では Result 型を使う
- Result 型と async/await を混在させない

## DO NOT
- `try!` を使わない
- `catch { }` で何もしない空キャッチを書かない
- エラーケースに `.failed` / `.error` のような曖昧な名前を付けない
