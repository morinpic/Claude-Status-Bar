## サイズの目安

| サイズ | 目安 |
|--------|------|
| S | 数時間以内。1〜2ファイルの変更 |
| M | 半日〜1日。複数ファイルにまたがる変更 |
| L | 数日。設計から必要な大きな機能追加 |

## 運用ルール

1. やりたいことが出たら、まずこのファイルに追記する
2. 優先度を決めたら GitHub Issue を作成し、Issue 列にリンクを記載する
3. 実装するときは Claude Code に Issue 番号を渡して指示する
4. 完了したら行ごと削除するか、取り消し線で対応済みと分かるようにする
5. BACKLOG.md の更新は実装作業とセットでコミットする（項目のまとめ追加時は単独コミット可）
