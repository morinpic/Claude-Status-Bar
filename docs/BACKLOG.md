## サイズの目安

| サイズ | 目安 |
|--------|------|
| S | 数時間以内。1〜2ファイルの変更 |
| M | 半日〜1日。複数ファイルにまたがる変更 |
| L | 数日。設計から必要な大きな機能追加 |

## 運用ルール

1. やりたいことが出たら、まずこのファイルに追記する
2. 実装するときはバックログ ID（F-13 等）でブランチを切って作業する
3. 完了したら取り消し線で対応済みと分かるようにする（行を削除しない）
4. BACKLOG.md の更新は実装作業とセットでコミットする（項目のまとめ追加時は単独コミット可）
5. GitHub Issue は外部報告・設計議論が必要な場合のみ作成し、Issue 列にリンクを記載する
