# UI/UX 設計仕様書
# Claude Status Bar — macOS メニューバー ステータス監視アプリ

> 設計哲学: **"Glanceable, Actionable, Non-intrusive"**
>
> メニューバーアプリの本質は「存在感を消しながら、必要な瞬間だけ前に出る」こと。
> ユーザーのフローを壊さない最小介入を原則とする。

---

## 1. 情報階層（3層構造）

```
Layer 1: メニューバーアイコン（常時表示）
│  ├─ 色 + 形状でステータスを瞬時に伝える
│  ├─ 視認所要時間: <100ms
│  └─ 情報密度: 最小（1bit的判断 = 正常 or 異常）
│
Layer 2: ポップオーバー（クリック on demand）
│  ├─ 全コンポーネントのステータス一覧
│  ├─ 直近インシデントのサマリー
│  └─ 設定へのアクセス
│
Layer 3: 詳細（ポップオーバー内展開 / 外部リンク）
   ├─ インシデントカードの展開表示
   └─ status.claude.com への深リンク
```

**設計根拠:** HIG の "Progressive Disclosure" 原則。常時全情報を見せると認知負荷が上がり、アイコンを見る頻度が下がる。3層にすることで各層の信号対雑音比を最大化。

## 2. メニューバーアイコン（4状態）

### アイコン仕様

| プロパティ | 値 |
|---|---|
| サイズ | 16×16pt（@2x: 32px） |
| スタイル | SF Symbols ベース + カスタム |
| 余白 | 左右各 4pt（macOS 標準マージン） |
| アニメーション | Core Animation / 60fps |

### 状態別デザイン

| 状態 | アイコン形状 | 色（Light / Dark） |
|---|---|---|
| 正常 | ● 塗り円（claude.ai "C" モチーフ） | #34C759 / #30D158 |
| 警告 | ◐ 半塗り円（右半分が欠けた状態） | #FF9500 / #FF9F0A |
| 障害 | ✕ バツ付き円（円 + 斜線 2本） | #FF3B30 / #FF453A |
| 不明/ローディング | ◌ アウトライン円（点滅アニメーション） | #8E8E93 / #636366 |

### ステータス変化アニメーション（マイクロインタラクション）

**正常 → 警告**
- duration: 0.4s
- timing: easeInOut
- effect: 色クロスフェード + 軽いスケール pulse (1.0 → 1.15 → 1.0)

**正常/警告 → 障害**
- duration: 0.3s
- timing: easeIn（緊急性を演出）
- effect: 色変化 + バツ印がストロークとして描画される（draw-on）

**障害 → 正常（復旧）**
- duration: 0.6s
- timing: spring(damping: 0.7)
- effect: バツ消去 → 円が再描画 → 緑にフェード（「治った」感）

**ローディング中**
- effect: opacity 0.4 ↔ 1.0 のパルス（1.2s周期）
- ※ CPU 節約のため CABasicAnimation を使用、バックグラウンド時は停止

### メニューバー空きスペース対応（レスポンシブ）

| 空きスペース状況 | 表示内容 |
|---|---|
| 余裕あり | アイコン（16pt） |
| 狭い | アイコン（12pt に縮小） |
| 極端に狭い | アイコンを最右端に移動（macOSが自動制御） |

> MenuBarExtra の label に `.labelStyle(.iconOnly)` を指定。テキストラベルは表示しない（スペース節約）。

## 3. ポップオーバー — レイアウト仕様

### 基本寸法

| プロパティ | 値 |
|---|---|
| 幅 | 300pt（固定） |
| 高さ | 動的（最小 160pt、最大 520pt） |
| 角丸 | システム標準（macOS Sonoma: 10pt） |
| 内側余白 | 上下 12pt、左右 16pt |

### 共通ヘッダー

```
┌──────────────────────────────────────────┐  ← 幅300pt
│  ⬤ Claude Status          ↻ 更新中...  │  24pt
│  最終更新: 2分前                          │  16pt
└──────────────────────────────────────────┘
```

- 左: ステータスドット（大・12pt）+ アプリ名（.headline）
- 右: 最終更新時刻（.caption2、secondaryLabel色）
- ポーリング中は右端に ActivityIndicator（8pt）

### 画面 6: 通知バナー

macOS 標準通知センター形式（UNUserNotificationCenter）

```
┌─────────────────────────────────────┐
│ [アプリアイコン]  Claude Status Bar   │
│                  障害を検知          │
│ Claude API で障害が発生しました。     │
│ 影響: Claude API · 重大             │
│                          12:34      │
└─────────────────────────────────────┘
```

**通知設計原則（ノイズ回避）:**

通知を送るべき状況:
- ✅ operational → degraded/outage への遷移（悪化）
- ✅ outage → operational への遷移（復旧）

通知しない状況:
- ❌ degraded → degraded（同レベルの更新）
- ❌ ポーリング失敗1回目（指数バックオフ後に再試行）
- ❌ アプリ起動直後の初回取得結果

**設計根拠:** ペルソナの最大の課題は「通知ノイズ」。誤った通知が1回でも来ると、以降の通知を無視するようになる（通知疲れ）。State diff による厳密なトリガー制御が信頼性の鍵。

## 5. ステータスバッジ コンポーネント

### ドット仕様

```swift
// ドットサイズと色の仕様
enum StatusBadge {
    case operational  // ●  8pt  #34C759
    case degraded     // ◐  8pt  #FF9500
    case outage       // ✕  8pt  #FF3B30  (丸 + バツアイコン)
    case unknown      // ◌  8pt  #8E8E93  (アウトラインのみ)
}

// VoiceOver ラベル（アクセシビリティ）
func accessibilityLabel(_ status: StatusBadge) -> String {
    switch status {
    case .operational: return "正常稼働中"
    case .degraded:    return "パフォーマンス低下中"
    case .outage:      return "サービス障害発生中"
    case .unknown:     return "ステータス不明"
    }
}
```

### コントラスト比確認

**Light Mode:**
- ● 緑 #34C759 on White → コントラスト比 2.8:1（ドットのみ、テキストは黒）
- テキスト "正常" #000000 → 21:1 ✅

**Dark Mode:**
- ● 緑 #30D158 on #1C1C1E → コントラスト比 5.1:1 ✅
- テキスト "正常" #FFFFFF → 16:1 ✅

> ※ カラーのみに依存しない設計: アイコン形状・ラベルテキストを必ず併記。色盲ユーザー向けに形状で状態を区別（● / ◐ / ✕ / ◌）。WCAG 2.1 Success Criterion 1.4.1 準拠。

## 6. ナビゲーション設計

### ジェスチャーマッピング

| 操作 | アクション |
|---|---|
| 左クリック | ポップオーバーを開く/閉じる |
| 右クリック | コンテキストメニュー（今すぐ更新 / 設定を開く / 終了） |
| Cmd+クリック | 予約（将来の機能拡張用） |
| Option+クリック | デバッグ情報表示（開発者向け） |
| クリック（リンク） | 外部ブラウザで status.claude.com を開く |
| Escape | ポップオーバーを閉じる |
| スクロール | コンポーネント一覧のスクロール |

**設計根拠:** 右クリックにクイックアクションを集約することで、「設定を開く→終了する」という頻度の高い操作を2ステップから1ステップに短縮。

## 7. アクセシビリティ

### VoiceOver

```swift
// メニューバーアイコン
.accessibilityLabel("Claude Status Bar")
.accessibilityValue("全サービス正常稼働中")
// 障害時:
.accessibilityValue("Claude API で障害発生中。影響: Claude API、Claude Code")

// ポップオーバーのフォーカス順
// 1. ヘッダー（全体ステータス）
// 2. インシデントカード（存在する場合）
// 3. コンポーネント行（上から順）
// 4. フッターリンク群
```

### Dynamic Type

ポップオーバー幅 300pt は固定だが、フォントサイズに追従:
- AccessibilityExtraExtraLarge 以上ではコンポーネント行を2行に折り返し（サービス名 / ステータス）
- 設定UI ではグリッドを3列→2列に自動調整

### Reduce Motion

```swift
// アイコンアニメーション
if !accessibilityReduceMotion {
    // パルスアニメーション実行
} else {
    // 即座に色変更のみ（アニメーションなし）
}
```

## 8. ポップオーバートランジション

**表示:**
- duration: 0.2s
- effect: opacity 0→1 + translateY(-4pt→0)（上から降りてくる）
- timing: easeOut

**非表示:**
- duration: 0.15s（表示より短く：「閉じる」は速い方がよい）
- effect: opacity 1→0
- timing: easeIn

> macOS標準に近いため、カスタムは最小限。MenuBarExtra のデフォルトトランジションを尊重。

## 10. デザイナーズノート — 主要判断の根拠

| 判断 | 根拠 |
|---|---|
| ポップオーバー幅300pt固定 | HIG推奨範囲(250-350pt)の中央値。狭すぎると読めず、広いと視線移動コスト増大 |
| 障害時インシデントを最上部に | F字スキャンパターン。ユーザーは左上から読むため、最重要情報を最上部に |
| 正常時はインシデントカード非表示 | 「異常がないこと」を伝えるのに余分な空白は不要。情報の欠如が正常を意味する |
| 前回成功ステータスの保持表示 | ネットワーク断でアプリが無価値になるのを防ぐ。最後の既知状態が意思決定に役立つ |
| 右クリックコンテキストメニュー | macOSパワーユーザーは右クリック起点。設定・終了へのショートカットとして機能 |
| 通知はState diffのみでトリガー | 毎ポーリングで通知すると信頼性ゼロになる。状態「変化」のみが通知に値する情報 |
| Skeleton UIをローディングに使用 | スピナーより体感速度が改善。コンテンツの「場所」を予告することで安心感を提供 |
| 形状+色でステータスを区別 | 色覚多様性ユーザー対応。WCAG 2.1 Success Criterion 1.4.1準拠 |
| フッターをツールバー形式に | ポップオーバーの縦スペースを節約。設定・リンクを一行に集約 |
| アニメーション duration 0.2-0.4s | HIG推奨の「瞬間的」アニメーション範囲。長すぎると操作の邪魔になる |
